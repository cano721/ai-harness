# QA Team - CLAUDE.md

## 도메인
ATS (Applicant Tracking System) 채용 관리 시스템

## 테스트 시나리오 작성

수용 기준(Acceptance Criteria) 기반으로 작성한다.

### 형식

```
시나리오: [시나리오 이름]

Given [사전 조건/상태]
When  [사용자 행동/이벤트]
Then  [기대 결과]
```

## ATS 핵심 E2E 시나리오

채용 전체 플로우:
```
채용공고 등록 → 지원 → 서류심사 → 면접 → 합격 → 입사
```

핵심 시나리오는 E2E 커버리지 100% 달성 필수.

## 테스트 도구

| 레이어 | 도구 |
|--------|------|
| E2E | Playwright |
| API | RestAssured |
| 단위/통합 | 개발팀 기준 따름 |

## 커버리지 기준

| 레이어 | 목표 |
|--------|------|
| 단위 테스트 | 80% |
| 통합 테스트 | 60% |
| E2E (핵심 시나리오) | 100% |

## 테스트 원칙

- 자동 테스트 우선
- 수동 테스트는 탐색적 테스트(Exploratory Testing)에 한정
- 회귀 테스트 범위는 변경 영향도에 비례하여 결정

## 테스트 데이터

- 팩토리 패턴으로 테스트 데이터 생성
- 테스트 간 데이터 격리 필수 (상호 영향 금지)
- 프로덕션 데이터 테스트 환경 사용 금지

```java
// 팩토리 예시
ApplicantFactory.create()
  .withName("홍길동")
  .withStatus(ApplicantStatus.DOCUMENT_REVIEW)
  .build();
```

## 버그 리포트 형식

```
제목: [컴포넌트] 버그 요약

환경: (개발/스테이징/프로덕션)
심각도: (Critical/High/Medium/Low)

재현 단계:
1. ...
2. ...

기대 동작: ...
실제 동작: ...

첨부: 스크린샷/로그
```

## 릴리즈 게이팅

- 핵심 E2E 시나리오 통과 없이 프로덕션 배포 불가
- Critical/High 버그 오픈 상태에서 릴리즈 불가
