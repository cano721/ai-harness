# 테스트 시나리오 생성 Skill

## 사용법
/harness test-scenario [유저 스토리 또는 수용 기준]

예시: `/harness test-scenario "지원자가 이력서를 첨부하여 지원서를 제출한다"`
예시: `/harness test-scenario "AC: 채용담당자는 지원자 상태를 변경할 수 있어야 한다"`

## 실행 내용

입력받은 유저 스토리 또는 수용 기준을 분석하여 ATS 도메인 컨텍스트에 맞는 완전한 테스트 시나리오를 생성하라.

### ATS 도메인 주요 시나리오 카탈로그

**지원서 제출 시나리오**
- 정상: 이력서 첨부 후 모든 필수 항목 입력 → 지원서 제출 성공
- 예외: 이력서 미첨부 → 제출 차단 및 오류 메시지
- 예외: 지원 마감된 공고 → 제출 불가 안내
- 경계: 파일 크기 최대치(10MB)로 업로드
- 경계: 동일 공고에 중복 지원 시도

**면접 평가 시나리오**
- 정상: 면접관이 모든 항목에 점수 입력 후 제출
- 예외: 평가 기한 초과 후 제출 시도
- 예외: 담당 면접이 아닌 건 평가 시도
- 경계: 점수 최솟값(0점) 및 최댓값(100점) 입력

### 자동화 가능 여부
각 시나리오에 대해 E2E 자동화 가능 여부를 판단하고 표시하라.
- 자동화 권장: UI 인터랙션이 명확하고 반복 실행 가능한 경우
- 수동 테스트 권장: 주관적 판단이 필요하거나 외부 시스템 의존성이 높은 경우
