# Planning Team - CLAUDE.md

## 도메인
ATS (Applicant Tracking System) 채용 관리 시스템

## PRD 작성 규칙

모든 PRD는 다음 섹션을 필수로 포함한다:

- **배경**: 이 기능이 필요한 이유, 현재 문제점
- **목적**: 이 기능으로 달성하고자 하는 목표
- **범위 (In-scope)**: 이번 PRD에서 다루는 기능
- **비범위 (Out-of-scope)**: 명시적으로 제외하는 기능

## 유저 스토리 형식

```
As a [채용담당자 | 지원자 | 면접관],
I want [기능/행동],
so that [얻는 가치/목적].
```

## 수용 기준 (Acceptance Criteria)

Given-When-Then 형식으로 작성한다:

```
Given [사전 조건/상태]
When [사용자 행동/이벤트]
Then [기대 결과]
```

## 채용 도메인

### 채용 프로세스 플로우
```
지원완료 → 서류심사 → 면접 → 합격/불합격
```

### 주요 엔티티
- **JobPosting**: 채용공고 (직무, 자격요건, 마감일)
- **Applicant**: 지원자 (이력서, 지원 이력)
- **Interview**: 면접 (일정, 면접관, 유형)
- **Evaluation**: 평가 (면접관별 평가, 점수, 의견)
- **Offer**: 합격 통보 (처우, 입사일)

### 주요 페르소나
- **채용담당자**: 채용공고 관리, 지원자 심사, 프로세스 조율
- **지원자**: 공고 검색, 지원, 전형 상태 확인
- **면접관**: 면접 일정 확인, 평가 제출

## 기획 원칙

- 기술 구현 방식을 지정하지 않는다 (개발팀 자율)
- 비즈니스 요구사항과 사용자 가치에 집중한다
- 모호한 요구사항은 예시와 엣지케이스로 명확히 한다
- 우선순위: MoSCoW (Must / Should / Could / Won't)

## 발행 규칙

- PRD 작성 완료 후 Confluence에 발행 필수
- 리뷰어: PM, 개발팀 리드, QA 리드
- 승인 후 개발팀에 공유
