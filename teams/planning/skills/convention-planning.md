# Planning 문서 컨벤션

## PRD 필수 섹션
모든 PRD는 다음 섹션을 포함한다:

| 섹션 | 설명 |
|------|------|
| 배경 | 이 기능이 필요한 이유, 현재 문제점 |
| 목적 | 달성하고자 하는 목표 |
| 범위 (In-scope) | 이번 PRD에서 다루는 기능 |
| 비범위 (Out-of-scope) | 명시적으로 제외하는 기능 |
| 타겟 사용자 | 주요 페르소나 |
| 유저 스토리 | As a / I want / so that |
| 수용 기준 | Given-When-Then |
| 기술 제약 | 개발팀이 고려해야 할 제약 (구현 방식 지정 금지) |
| 일정 | 마일스톤 |

## 유저 스토리 형식
```
As a [채용담당자 | 지원자 | 면접관 | HR관리자],
I want [기능/행동],
so that [얻는 가치/목적].
```

## 수용 기준 (Acceptance Criteria)
Given-When-Then 형식, 정상/예외/경계값 포함:

```
Given [사전 조건/상태]
When  [사용자 행동/이벤트]
Then  [기대 결과]
```

## 주요 페르소나
- **채용담당자**: 채용공고 관리, 지원자 심사, 프로세스 조율
- **지원자**: 공고 검색, 지원, 전형 상태 확인
- **면접관**: 면접 일정 확인, 평가 제출
- **HR관리자**: 전체 채용 현황 관리, 리포팅

## 채용 프로세스 플로우
```
지원완료 → 서류심사 → 면접 → 합격/불합격
```

## 우선순위 기준
MoSCoW:
- **Must**: 반드시 있어야 하는 기능
- **Should**: 있으면 좋은 기능
- **Could**: 여유가 있으면 추가
- **Won't**: 이번 범위에서 제외

## 발행 규칙
- PRD 작성 완료 후 Confluence에 발행 필수
- 리뷰어: PM, 개발팀 리드, QA 리드
- 승인 후 개발팀에 공유
