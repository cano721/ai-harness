# AI Harness - 개요

## AI 하네스란?

AI 하네스(Harness)는 AI 에이전트(Agent)가 자율적으로 업무를 수행할 때 **안전하게 통제하고 효율을 높이도록 감싸는 제어 및 검증 시스템(프레임워크)**이다.

모델의 성능보다 AI가 작동하는 환경 설계가 더 중요해지면서, AI가 실수를 줄이고 신뢰성 있게 장기 작업을 수행하도록 돕는 안전장치 역할을 한다.

## 적용 대상

- Claude Code, Codex 등 AI 코딩 에이전트
- 서비스 개발 전 과정: 기획, 디자인, 프론트엔드, 백엔드

## 핵심 목표

| 목표 | 설명 |
|------|------|
| **안전성** | 위험 명령 차단, 프로덕션 보호, 민감 정보 유출 방지 |
| **일관성** | 회사 규칙/컨벤션을 모든 AI 작업에 자동 적용 |
| **효율성** | 반복 검증 자동화, 워크플로우 표준화 |
| **커스터마이징** | 팀별로 규칙/도구/워크플로우를 독립적으로 관리 |
| **추적성** | 모든 AI 액션을 로깅하여 감사/개선 가능 |

## 기존 도구와의 관계

```
AI Harness (회사 제어 레이어)
    ↕ 연동
OMC / OMX (에이전트 오케스트레이션)
    ↕ 연동
Claude Code / Codex (AI 에이전트)
    ↕ 호출
LLM (Claude, GPT 등)
```

하네스는 새 프레임워크를 처음부터 만드는 것이 아니라, **기존 OMC/OMX의 확장 포인트(Hook, MCP, Skill, Agent)를 활용하여 회사 전용 레이어를 구축**하는 방식이다.
