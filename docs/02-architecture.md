## 5대 구성요소

### 1. Context Layer (CLAUDE.md 체계)

AI 에이전트가 작업 시 자동으로 참조하는 규칙/지침 문서 체계.

- **Global CLAUDE.md**: 회사 공통 코딩 원칙, 보안 정책, 금지 패턴
- **Team CLAUDE.md**: 팀별 컨벤션, 기술 스택 규칙, 도메인 용어
- **Project CLAUDE.md**: 프로젝트 특화 규칙 (기존 방식 그대로)

적용 우선순위: `Project > Team > Global` (하위가 상위를 오버라이드)

### 2. Hooks (실행 전후 자동 검증)

모든 AI 액션에 대해 자동으로 실행되는 검증 스크립트.

| 시점 | 역할 | 예시 |
|------|------|------|
| **PreToolUse** | 실행 전 차단/경고 | 위험 명령 차단, 권한 체크, 프로덕션 DB 접근 차단 |
| **PostToolUse** | 실행 후 검증 | lint/format, 보안 스캔, 번들 사이즈 체크 |
| **Audit** | 모든 액션 로깅 | 누가/언제/무엇을 했는지 기록, 비용 추적 |
| **Notification** | 알림 발송 | 작업 완료, 에러 발생, 승인 요청 시 Teams/Slack 알림 |

### 3. MCP Servers (외부 도구 연동)

AI 에이전트가 접근할 수 있는 외부 시스템 연동 레이어.

- **공통**: Jira, Confluence, Git, Teams/Slack
- **팀별**: Figma(디자인), Storybook(FE), DB/모니터링(BE)

### 4. Workflow Templates (업무별 플레이북)

업무 유형별로 에이전트가 따라야 할 표준 프로세스 정의.

```
[기능 개발 워크플로우]

Jira 이슈 확인
    ↓
기획 검토 (Confluence 참조)
    ↓
디자인 확인 (Figma 참조)
    ↓
──── 승인 게이트 ① : 구현 범위 확인 ────
    ↓
코드 구현 (팀별 CLAUDE.md 규칙 적용)
    ↓
자동 검증 (테스트, lint, 보안 스캔)
    ↓
──── 승인 게이트 ② : 코드 리뷰 ────
    ↓
PR 생성 + 리뷰어 지정
    ↓
Teams 알림
```

### 5. Guardrails (안전장치)

AI 에이전트의 행동 범위를 제한하는 정책.

| 항목 | 설명 |
|------|------|
| 비용 한도 | 단일 작업당 최대 API 비용 제한 |
| 실행 시간 | 장시간 작업 자동 중단 |
| 파일 접근 | 수정 가능/불가 디렉토리 지정 |
| 프로덕션 보호 | prod 환경 직접 변경 차단 |
| 변경 범위 | 한 번에 수정 가능한 최대 파일 수 |

## 레이어 상속 모델

```
Global (회사 공통)
  │
  ├── 보안: 전사 필수 (오버라이드 불가)
  ├── 코딩 원칙: 기본값 제공 (팀에서 확장 가능)
  └── Hook: 공통 Hook 항상 실행 + 팀별 Hook 추가 실행

Team (팀별)
  │
  ├── 컨벤션: 팀 기술 스택에 맞는 규칙
  ├── Skill: 팀 업무에 특화된 커맨드
  └── Hook: 팀 전용 검증 로직 추가

Project (프로젝트별)
  │
  └── 기존 CLAUDE.md 방식 그대로 사용
```

**오버라이드 불가 항목** (Global에서 강제):
- 보안 정책 (민감 정보 유출 방지, 인젝션 방지)
- 감사 로깅 (비활성화 불가)
- 프로덕션 보호 정책
