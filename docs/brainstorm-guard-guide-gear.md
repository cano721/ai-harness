# Ddalkak ProjectDetail — Guard/Guide/Gear 개선 브레인스토밍

## Objective
ProjectDetail 페이지에서 Guard(안전), Guide(품질), Gear(효율) 3축이 실질적으로 동작하도록 개선한다.

## Current State
- Guard: Hook 적용/삭제만 가능. 차단 이벤트는 별도 Security 페이지에서만 보임.
- Guide: 컨벤션/스킬 파일 관리만. 에이전트가 실제로 따르는지 추적 불가.
- Gear: 비용/태스크 숫자만 표시. 프로젝트에서 직접 태스크 실행 UX 없음.

---

## PM 관점 (비즈니스 가치 + 고객 영향)

### Guard
1. **프로젝트별 보안 대시보드** — ProjectDetail에 최근 차단 이벤트 3~5개를 인라인 표시. "이 프로젝트에서 오늘 rm -rf 2회 차단됨" 식으로 즉시 파악.
2. **차단 알림 뱃지** — 프로젝트 카드와 사이드바에 미확인 차단 이벤트 수 뱃지. 확인 전까지 유지.
3. **Hook 테스트** — "테스트 실행" 버튼으로 Hook이 실제 동작하는지 시뮬레이션. 셋업 후 "정말 차단되는가?" 불안 해소.
4. **차단 트렌드** — 주간 차단 횟수 추이 미니 차트. 차단이 늘고 있으면 에이전트 사용 패턴에 문제가 있다는 신호.
5. **위험도 스코어** — Guard 셋업 상태를 점수화 (Hook 적용 여부, CLAUDE.md 보안 규칙 포함 여부). "보안 점수: 80/100"

### Guide
6. **컨벤션 준수율 추적** — 에이전트 실행 후 생성된 코드에서 컨벤션 위반 횟수를 기록. "최근 10회 실행 중 컨벤션 준수율 92%"
7. **CLAUDE.md 품질 체크** — CLAUDE.md 내용을 분석하여 빠진 섹션 안내. "기술 스택 섹션이 없습니다", "보안 규칙이 없습니다"
8. **컨벤션 추천** — 프로젝트 코드를 분석하여 현재 사용 중인 패턴을 감지하고 컨벤션으로 추천. "이 프로젝트에서 arrow function을 95% 사용 중 → 컨벤션으로 등록할까요?"
9. **스킬 효과 측정** — 스킬 적용 전/후 에이전트 출력 품질 비교. 스킬이 실제로 도움이 되는지 데이터로 확인.
10. **팀 컨벤션 동기화** — 여러 프로젝트에서 공통 컨벤션을 "글로벌 컨벤션"으로 승격. 한곳에서 수정하면 전체 반영.

### Gear
11. **프로젝트 내 태스크 실행** — ProjectDetail에서 바로 "에이전트에게 지시" 입력 → 태스크 생성 + 실행 + 로그 인라인 표시.
12. **에이전트 추천** — 태스크 내용을 분석하여 최적 에이전트 추천. "이 태스크는 Claude Code가 적합합니다 (코드 수정 포함)"
13. **비용 예산 설정** — 프로젝트별 월간 비용 한도. 초과 시 경고 또는 실행 차단.
14. **실행 히스토리 타임라인** — 프로젝트의 에이전트 실행 이력을 타임라인으로 시각화. 언제 무엇을 실행했고 결과가 무엇이었는지.
15. **효율 리포트** — 주간/월간 프로젝트 효율 요약 자동 생성. "이번 주 12개 태스크 완료, 평균 $0.05/태스크, 성공률 91%"

## Designer 관점 (UX + 사용성 + 만족감)

### Guard
1. **Guard 상태 표시줄** — ProjectDetail 상단에 초록/노랑/빨강 상태바. Hook 전부 적용 = 초록, 일부 = 노랑, 없음 = 빨강. 한눈에 보안 상태 파악.
2. **차단 이벤트 토스트** — 실시간 차단 발생 시 ProjectDetail 우측 하단에 토스트 알림. "방금 secret-scanner가 API 키 노출을 차단했습니다"
3. **Hook 시각화** — 어떤 Hook이 어떤 도구를 감시하는지 플로우 다이어그램. "Bash 명령 → block-dangerous → 차단/허용"
4. **보안 체크리스트** — Guard 섹션을 체크리스트 형태로. ✓ 위험 명령 차단, ✓ 시크릿 스캔, ✗ 인프라 보호. 미완료 항목 클릭 시 바로 적용.
5. **차단 상세 드로어** — 차단 이벤트 클릭 시 우측에서 드로어 슬라이드. 차단 사유, 원본 명령, 대안 제시를 한 화면에.

### Guide
6. **셋업 위자드** — 처음 프로젝트 진입 시 스텝바이스텝 위자드. "1/4 CLAUDE.md → 2/4 Hook → 3/4 컨벤션 → 4/4 완료". 원클릭보다 과정이 보여서 안심.
7. **컨벤션 카테고리 아이콘** — naming은 Aa, style은 {}. structure는 📁. 시각적 구분으로 빠른 스캔.
8. **CLAUDE.md 미리보기 분할뷰** — 수정 시 좌측 에디터 + 우측 마크다운 렌더링 미리보기. 실시간 프리뷰.
9. **스킬 카드 UI** — 스킬을 뱃지가 아닌 카드로. 이름 + 한줄 설명 + 아이콘. 클릭하면 전체 내용.
10. **가이드 완성도 게이지** — "Guide 셋업 75%" 원형 프로그레스. CLAUDE.md ✓, 컨벤션 ✓, 스킬 ✗ → 75%.

### Gear
11. **인라인 태스크 입력** — 페이지 하단에 고정된 입력바. "에이전트에게 지시..." 입력 → Enter → 태스크 생성 + 실행. 채팅처럼 자연스러운 UX.
12. **실행 로그 패널** — 태스크 실행 시 하단에서 터미널 패널이 올라옴. SSE로 실시간 로그. 완료 시 결과 요약.
13. **비용 미니 차트** — 비용 섹션에 최근 7일 스파크라인 차트. 숫자만 보는 것보다 추세 파악이 쉬움.
14. **에이전트 상태 애니메이션** — running 에이전트에 pulse 애니메이션. 시각적으로 "일하고 있다" 피드백.
15. **빈 상태 일러스트** — 각 섹션의 빈 상태에 적절한 일러스트 + 한 줄 설명. 차가운 "No data" 대신 따뜻한 안내.

## Engineer 관점 (기술 가능성 + 데이터 활용)

### Guard
1. **Hook 드라이런** — 테스트 명령을 입력하면 Hook 파이프라인을 시뮬레이션. 실제 실행 없이 "이 명령이면 차단됩니다" 확인. 기존 hook 테스트 스크립트 재활용.
2. **차단 이벤트 WebSocket** — SSE/WebSocket으로 실시간 차단 이벤트 푸시. 페이지 새로고침 없이 즉시 표시.
3. **Hook 체인 검증** — 등록된 Hook들의 실행 순서와 충돌 여부를 자동 검증. "block-dangerous와 infra-change-review가 같은 패턴을 중복 검사합니다"
4. **settings.json diff** — Hook 변경 시 before/after diff 표시. 무엇이 바뀌는지 확인 후 적용.
5. **Hook 실행 로그** — 각 Hook의 실행 횟수, 평균 실행 시간, 차단률 통계. 느린 Hook 감지.

### Guide
6. **CLAUDE.md 린터** — CLAUDE.md를 파싱하여 필수 섹션(기술스택, 컨벤션, 보안규칙) 누락 체크. JSON schema처럼 구조 검증.
7. **컨벤션 → Hook 자동 변환** — "매직넘버 금지" 컨벤션을 자동으로 정규식 기반 Hook으로 변환하여 실제 강제. Guide → Guard 연결.
8. **코드 패턴 분석기** — git diff나 최근 커밋을 분석하여 반복 패턴 감지 → 컨벤션 후보로 추천. 기존 코드에서 자동 학습.
9. **스킬 템플릿 라이브러리** — 기술 스택별 스킬 템플릿 제공. "React 프로젝트용 develop 스킬", "Spring Boot 리뷰 스킬". 셋업 시 자동 선택.
10. **convention.md 파서** — convention.md를 구조화된 데이터로 파싱. 카테고리별 그룹화, 검색, 필터 지원. 마크다운이지만 UI에서는 구조적으로 표시.

### Gear
11. **태스크 큐 시각화** — 프로젝트의 태스크 큐를 파이프라인으로 표시. todo → in_progress → done 칸반 보드.
12. **에이전트 자동 선택** — 태스크 내용을 분석하여 가장 적합한 에이전트 자동 매칭. "코드 리뷰" → Claude, "대량 리팩토링" → Codex.
13. **비용 예측** — 태스크 설명 기반으로 예상 비용/토큰 추정. 실행 전에 "약 $0.03, 2분 소요 예상" 표시.
14. **실행 결과 캐싱** — 동일 태스크 재실행 시 이전 결과 비교. diff 표시로 "지난번과 뭐가 달라졌는지" 확인.
15. **벤치마크 모드** — 같은 태스크를 Claude/Codex/Cursor 3개 에이전트로 동시 실행하여 결과/비용/속도 비교.

## Top 5 우선순위 아이디어

### 1. Guard 상태 표시줄 + 프로젝트별 차단 이벤트 인라인
**설명**: ProjectDetail 상단에 Guard 상태바(초록/노랑/빨강) + 최근 차단 이벤트 3개 인라인 표시.

**선정 이유**: Guard가 "설정만 하고 끝"이 아니라 "실시간으로 지켜보고 있다"는 느낌을 줌. 기존 데이터(activity_log)를 활용하므로 신규 인프라 불필요.

**검증할 가정**:
- 사용자가 ProjectDetail에서 차단 이벤트를 확인하고 싶어하는가? (vs Security 페이지로 이동)
- 상태 색상 3단계가 직관적인가?

---

### 2. 셋업 위자드 (스텝바이스텝)
**설명**: 원클릭 셋업을 스텝바이스텝 위자드로 확장. "1/4 CLAUDE.md → 2/4 Hook → 3/4 컨벤션 → 4/4 스킬". 각 단계에서 생성될 내용 미리보기 + 커스터마이즈 가능.

**선정 이유**: 원클릭은 편하지만 "뭘 만들었는지" 모르고 지나감. 위자드는 각 단계를 이해하면서 셋업할 수 있어 학습 효과 + 신뢰도 향상.

**검증할 가정**:
- 사용자가 각 단계를 커스터마이즈하고 싶어하는가? (vs 그냥 원클릭)
- 위자드가 "귀찮다"고 느끼지는 않는가?

---

### 3. 인라인 태스크 입력 + 실행 로그 패널
**설명**: ProjectDetail 하단에 "에이전트에게 지시..." 입력바. 입력 → 태스크 생성 → 에이전트 실행 → 하단 터미널 패널에서 SSE 실시간 로그. 완료 시 결과 요약.

**선정 이유**: Gear의 핵심 — "프로젝트 컨텍스트 안에서 바로 에이전트에게 일을 시키는 것". 현재는 Tasks 페이지로 이동해야 하는데, 프로젝트 상세에서 바로 실행하는 게 자연스러움.

**검증할 가정**:
- 사용자가 대시보드에서 태스크를 실행하는가? (vs 터미널에서 직접 claude 실행)
- SSE 로그가 유용한가? (vs 결과만 보면 되는가)

---

### 4. CLAUDE.md 품질 체크 + 필수 섹션 안내
**설명**: CLAUDE.md 내용을 분석하여 빠진 섹션 안내. "보안 규칙 섹션이 없습니다 — 추가하시겠습니까?". 추가 클릭 시 해당 섹션 템플릿 자동 삽입.

**선정 이유**: CLAUDE.md가 Guide의 핵심인데, 내용이 부실하면 에이전트가 컨텍스트를 제대로 못 잡음. 품질 체크로 "좋은 CLAUDE.md"로 가이드.

**검증할 가정**:
- "필수 섹션"의 정의가 프로젝트마다 다르지 않은가?
- 자동 삽입된 템플릿이 실제로 유용한가?

---

### 5. Guard/Guide/Gear 완성도 점수
**설명**: 프로젝트 개요 섹션에 3축 완성도를 점수로 표시.
```
Guard 80%  ████████░░  Hook ✓, 시크릿스캔 ✓, 인프라보호 ✗
Guide 60%  ██████░░░░  CLAUDE.md ✓, 컨벤션 ✓, 스킬 ✗
Gear  40%  ████░░░░░░  에이전트 ✓, 태스크 ✗, 비용추적 ✗
```

**선정 이유**: "지금 이 프로젝트가 얼마나 잘 관리되고 있는가"를 한눈에. 게이미피케이션 효과로 셋업 동기 부여.

**검증할 가정**:
- 점수 기준이 객관적인가? (어떤 항목이 몇 %인지)
- 점수가 사용자에게 동기 부여가 되는가? (vs 무시하는가)
