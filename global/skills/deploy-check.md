# 배포 전 체크리스트 Skill

## 사용법
/harness deploy-check [배포 환경] [선택: Jira 이슈번호 또는 PR 번호]

예시: `/harness deploy-check production ATS-456`
예시: `/harness deploy-check staging PR-123`

## 실행 내용

지정된 배포 환경과 변경 내용을 분석하여 배포 전 완료해야 할 체크리스트를 생성하라. Jira 이슈나 PR이 제공된 경우 변경 내용을 조회하여 항목을 구체화하라.

---

## 출력 형식: 배포 전 체크리스트

**환경**: {environment}
**배포 버전**: {version}
**배포 담당자**: ___________
**배포 예정 시각**: ___________
**예상 소요 시간**: ___________
**롤백 기준 시간**: 배포 후 15분 내 이상 감지 시 즉시 롤백

---

### 1. 코드 & 빌드 검증

- [ ] CI/CD 파이프라인 전체 통과 확인
- [ ] 단위 테스트 100% 통과
- [ ] 통합 테스트 통과
- [ ] 코드 리뷰 승인 완료 (최소 1명 이상)
- [ ] 보안 스캔 (SAST/Dependency) 경고 없음
- [ ] 아티팩트 빌드 성공 및 레지스트리 푸시 확인
- [ ] 환경 변수 및 시크릿 최신화 여부 확인

### 2. DB 마이그레이션

- [ ] Flyway 마이그레이션 스크립트 존재 여부 확인
- [ ] 스테이징 환경에서 마이그레이션 사전 실행 및 검증
- [ ] 마이그레이션 소요 시간 측정 (대용량 테이블 주의)
- [ ] 롤백 스크립트(Undo) 준비 완료
- [ ] 마이그레이션 중 서비스 중단 여부 확인
- [ ] 인덱스 생성 포함 시 잠금(Lock) 영향 분석 완료

### 3. 캐시 워밍 & 무효화

- [ ] 변경된 API/데이터에 해당하는 캐시 무효화 계획 수립
- [ ] 캐시 워밍 스크립트 준비 (필요 시)
- [ ] Redis 키 패턴 변경 시 기존 키 정리 계획 확인
- [ ] CDN 캐시 퍼지(Purge) 대상 URL 목록 준비

### 4. 모니터링 & 알림 설정

- [ ] Datadog/CloudWatch 대시보드 접속 준비
- [ ] 배포 시간대 알림 임계값 조정 (오탐 방지)
- [ ] 에러율 모니터링 알림 활성화 확인
- [ ] 응답 시간 SLO 모니터링 확인
- [ ] 핵심 비즈니스 메트릭 알림 설정 (지원서 제출, 면접 등록 등)
- [ ] 온콜 담당자 사전 통보 완료

### 5. 롤백 계획

- [ ] 이전 버전 이미지/아티팩트 태그 확인: `{previous-version}`
- [ ] 롤백 명령어 준비 및 테스트 완료
- [ ] DB 롤백 필요 여부 및 방법 확인
- [ ] 롤백 시 캐시 처리 방법 확인
- [ ] 롤백 의사결정 기준 합의 완료
  - 에러율 > 5% 지속 시
  - P99 응답 시간 > 3초 지속 시
  - 핵심 기능(지원서 제출) 장애 시

### 6. 이해관계자 커뮤니케이션

- [ ] 배포 공지 Teams/Slack 발송 완료
- [ ] 영향 받는 팀(CS, 영업 등) 사전 통보
- [ ] 배포 완료 후 공지 템플릿 준비

### 7. 배포 후 즉시 확인 (Go/No-Go)

배포 완료 후 5분 내 확인:
- [ ] `GET /actuator/health` → UP
- [ ] 에러율 정상 범위 (< 1%)
- [ ] 핵심 API 응답 시간 정상
- [ ] 신규 기능 동작 확인 (스모크 테스트 1회)

- 인프라팀: ☐ 승인 / ☐ 보류
