# 스모크 테스트 체크리스트 생성 Skill

## 사용법
/harness smoke-test [배포 환경] [선택: 배포 버전]

예시: `/harness smoke-test staging v1.5.0`
예시: `/harness smoke-test production`

## 실행 내용

지정된 배포 환경에 맞는 ATS 스모크 테스트 체크리스트를 생성하라. 각 항목은 즉시 실행 가능한 구체적인 검증 방법을 포함해야 한다.

### 환경 구분
| 환경 | URL 패턴 | 검증 강도 |
|------|---------|---------|
| dev | dev-ats.company.com | 기본 |
| staging | staging-ats.company.com | 전체 |
| production | ats.company.com | 전체 + 모니터링 확인 |

## 출력 형식: 스모크 테스트 체크리스트

**환경**: {environment}
**버전**: {version}
**실행자**: ___________
**실행 일시**: ___________

---

### 1. 인프라 & 헬스체크

- [ ] **애플리케이션 헬스**: `GET /actuator/health` → `{"status": "UP"}` 응답 확인
- [ ] **데이터베이스 연결**: `GET /actuator/health/db` → status UP 확인
- [ ] **Redis 연결**: `GET /actuator/health/redis` → status UP 확인 (캐시 사용 시)
- [ ] **외부 연동 서비스**: `GET /actuator/health/external` → 모든 외부 서비스 UP 확인
- [ ] **서버 응답 시간**: 헬스체크 API 응답 200ms 이내
- [ ] **SSL 인증서**: HTTPS 접속 및 인증서 유효기간 30일 이상 확인
- [ ] **메모리 사용률**: 80% 미만 (Datadog/CloudWatch 확인)
- [ ] **CPU 사용률**: 70% 미만 (배포 직후 10분 모니터링)

### 2. 인증 & 권한

- [ ] **로그인 (채용담당자)**: 이메일/비밀번호 로그인 → 대시보드 이동
- [ ] **로그인 (지원자)**: 소셜 로그인 또는 이메일 로그인
- [ ] **로그아웃**: 로그아웃 후 세션 무효화 확인
- [ ] **인증 만료**: 토큰 만료 후 401 응답 + 재로그인 유도
- [ ] **권한 제어**: 지원자 계정으로 채용담당자 전용 페이지 접근 차단

### 3. 핵심 기능 동작

**채용공고 관리**
- [ ] 채용공고 목록 조회 (페이징 포함)
- [ ] 채용공고 상세 조회
- [ ] 채용공고 등록 (임시저장 → 게시)

**지원서 관리**
- [ ] 지원서 목록 조회 (채용담당자)
- [ ] 지원서 제출 (지원자)
- [ ] 파일 업로드 (이력서 PDF)
- [ ] 지원서 상태 변경

**면접 관리**
- [ ] 면접 일정 등록
- [ ] 면접 일정 조회 (면접관)
- [ ] 평가 작성 및 제출

### 4. 데이터베이스 연결 및 정합성

- [ ] **쓰기 작업**: 테스트 데이터 생성 → DB 반영 확인
- [ ] **읽기 작업**: 생성한 데이터 목록 API로 조회 가능
- [ ] **트랜잭션**: 실패 케이스에서 롤백 확인
- [ ] **마이그레이션**: Flyway 마이그레이션 성공 여부 (`flyway_schema_history` 확인)

### 5. 외부 시스템 연동

- [ ] **이메일 발송**: 지원 완료 알림 이메일 수신 확인
- [ ] **파일 스토리지**: 이력서 업로드 후 S3/GCS 저장 확인
- [ ] **알림 서비스**: 면접 일정 등록 시 푸시/이메일 알림 발송
- [ ] **외부 채용 플랫폼**: 채용공고 연동 API 응답 정상

### 6. 성능 기준선

- [ ] 채용공고 목록 API 응답 시간: 500ms 이내
- [ ] 지원서 목록 API 응답 시간: 800ms 이내
- [ ] 파일 업로드 (5MB): 3초 이내
- [ ] 대시보드 초기 로딩: 2초 이내
